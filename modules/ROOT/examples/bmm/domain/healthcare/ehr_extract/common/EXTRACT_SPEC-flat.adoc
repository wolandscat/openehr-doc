3+^h|  *class EXTRACT_SPEC*

h|*Description*
2+a|Specification of an Extract's contents. The specification consists of attributes
specifying the directory, a version specification (which versions of information items
are to be included) and a manifest (which entities are to be included in the extract).

h|
2+^h|*Feature Group "access"*

h|prop +
  0..1
|*version_spec*: xref:domain:domain-healthcare-ehr_extract.adoc#_class_extract_version_spec[EXTRACT_VERSION_SPEC]
a|Specification of which versions of information items to include in the Extract.
If Void, the default is latest versions only.

h|prop +
  
|*manifest*: xref:domain:domain-healthcare-ehr_extract.adoc#_class_extract_manifest[EXTRACT_MANIFEST]
a|Specification of entities (e.g. records) to include in the Extract.

h|prop +
  
|*extract_type*: xref:representation:representation-data_types.adoc#_class_dv_coded_text[DV_CODED_TEXT]
a|Coded term indicating the type of content required, e.g. openehr-ehr,
openehr-demographic, generic-emr.

h|prop +
  
|*include_multimedia*: xref:representation:representation-builtin_types.adoc#_class_boolean[Boolean]
a|Indicates whether in-line instances of DV_MULTIMEDIA in the source data are
included or not.

h|prop +
  
|*priority*: xref:representation:representation-builtin_types.adoc#_class_integer[Integer]
a|Requested priority of this request to be handled by server.

h|prop +
  
|*link_depth*: xref:representation:representation-builtin_types.adoc#_class_integer[Integer]
a|Degree of links to follow emanating from content items specified for inclusion.
0 = don't follow; 1 = follow first degree links; n = follow nth degree links.

h|prop +
  0..1
|*criteria*: xref:representation:representation-builtin_types.adoc#_class_list[List]<xref:representation:representation-data_types.adoc#_class_dv_coded_text[DV_CODED_TEXT]>
a|Queries specifying the contents of this Extract.

h|prop +
  0..1
|*other_details*: xref:representation:representation-data_structures.adoc#_class_item_structure[ITEM_STRUCTURE]
a|Other specification items. Archetypable.
