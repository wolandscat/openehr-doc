3+^h|  *class EXTRACT_ENTITY_MANIFEST*

h|*Description*
2+a|The manifest for one entity (e.g. EHR subject), identifying the entity and optionally
specifying top-level items to be included in the Extract. The extract_id_key attribute
is used to record the identifier used throughout the Extract for this entity.

h|
2+^h|*Feature Group "access"*

h|prop +
  
|*extract_id_key*: xref:representation:representation-builtin_types.adoc#_class_string[String]
a|Identifier by which this entity is known in the Extract.

h|prop +
  0..1
|*ehr_id*: xref:representation:representation-identification.adoc#_class_hier_object_id[HIER_OBJECT_ID]
a|EHR / EMR identifier for the entity at the target system.

h|prop +
  0..1
|*subject_id*: xref:representation:representation-identification.adoc#_class_party_ref[PARTY_REF]
a|Subject (i.e. patient or similar) identifier for the entity at the target system.

h|prop +
  0..1
|*other_ids*: xref:representation:representation-builtin_types.adoc#_class_list[List]<xref:representation:representation-identification.adoc#_class_object_ref[OBJECT_REF]>
a|Other identifiers that may be used to find the entity at the target system.

h|prop +
  0..1
|*item_list*: xref:representation:representation-builtin_types.adoc#_class_list[List]<xref:representation:representation-identification.adoc#_class_object_ref[OBJECT_REF]>
a|List of UIDs of items to be included in the Extract, in cases where individual
items are being requested by id.
