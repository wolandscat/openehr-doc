3+^h|  *class EHR_STATUS*

h|*Description*
2+a|Single object per EHR containing various EHR-wide status flags and settings, including
whether this EHR can be queried, modified etc. This object is always modifiable, in order
to change the status of the EHR as a whole.

NOTE: It is strongly recommended that the inherited attribute `uid` be populated in
`EHR_STATUS` objects, using the UID copied from the `object_id()` of the `uid` field
of the enclosing `VERSION` object. + For example, the `ORIGINAL_VERSION.uid`
`87284370-2D4B-4e3d-A3F3-F303D2F4F34B::uk.nhs.ehr1::2` would be copied to the `uid`
field of the `EHR_STATUS` object.

h|*Inherit*
2+|xref:representation:representation-data_marking.adoc#_class_locatable[LOCATABLE]

h|
2+^h|*Feature Group "access"*

h|prop +
  
|*subject*: xref:representation:representation-data_marking.adoc#_class_party_self[PARTY_SELF]
a|The subject of this EHR. The `external_ref` attribute can be used to contain a direct
reference to the subject in a demographic or identity service. Alternatively, the
association between patients and their records may be done elsewhere for security reasons.

h|prop +
  
|*is_queryable*: xref:representation:representation-builtin_types.adoc#_class_boolean[Boolean]
a|True if this EHR should be included in population queries, i.e. if this EHR is considered
active in the population.

h|prop +
  
|*is_modifiable*: xref:representation:representation-builtin_types.adoc#_class_boolean[Boolean]
a|True if the EHR, other than the `EHR_STATUS` object, is allowed to be written to. The
`EHR_STATUS` object itself can always be written to.

h|prop +
  0..1
|*other_details*: xref:representation:representation-data_structures.adoc#_class_item_structure[ITEM_STRUCTURE]
a|Any other details of the EHR summary object, in the form of an archetyped
`ITEM_STRUCTURE`.

2+a|
