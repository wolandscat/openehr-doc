3+^h| abstract  *class BMM_TYPE*

h|*Description*
2+a|Abstract idea of specifying a type in some context. This is not the same as 'defining' a
class. A type specification is essentially a reference of some kind, that defines the type
of an attribute, or function result or argument. It may include generic parameters that
might or might not be bound. See subtypes.

h|*Inherit*
2+|xref:representation:builtin_types-gen.adoc#_class_any[Any]

h|
2+^h|*Feature Group "access"*

h| func
|*type_name* : xref:representation:builtin_types-gen.adoc#_class_string[String]
a|Formal string form of the type as per UML.

h| func
|*type_signature* : xref:representation:builtin_types-gen.adoc#_class_string[String]
a|Signature form of the type name, which for generics includes generic parameter constrainer
types E.g. `Interval<T:Ordered>`.

Defaults to the value of `type_name()`.

h| func
|*is_abstract* : xref:representation:builtin_types-gen.adoc#_class_boolean[Boolean]
a|If true, indicates a type based on an abstract class, i.e. a type that cannot be directly
instantiated.

h| func
|*is_primitive* : xref:representation:builtin_types-gen.adoc#_class_boolean[Boolean]
a|If True, indicates that a type based solely on primitive classes.

h| func
|*unitary_type* : xref:languages:bmm-gen.adoc#_class_bmm_unitary_type[BMM_UNITARY_TYPE]
a|Type with any container abstracted away; may be a formal generic type.

h| func
|*effective_type* : xref:languages:bmm-gen.adoc#_class_bmm_effective_type[BMM_EFFECTIVE_TYPE]
a|Type with any container abstracted away, and any formal parameter replaced by its
effective constraint type.

h| func
|*flattened_type_list* : xref:representation:builtin_types-gen.adoc#_class_list[List]<xref:representation:builtin_types-gen.adoc#_class_string[String]>
a|Completely flattened list of type names, flattening out all generic parameters.
