3+^h|  *class ARCHETYPE_TERMINOLOGY*

h|*Description*
2+a|Local terminology of an archetype. This class defines the semantics of the terminology of
an archetype.

h|*Inherit*
2+|xref:representation:builtin_types-gen.adoc#_class_any[Any]

h|
2+^h|*Feature Group "access"*

h|prop +
  
|*is_differential*: xref:representation:builtin_types-gen.adoc#_class_boolean[Boolean]
a|True if this terminology only contains terms relating to a differential specialisation of
the owning artefact, rather than a complete set.

h|prop +
  
|*original_language*: xref:representation:builtin_types-gen.adoc#_class_string[String]
a|Original language of the terminology, as set at artefact creation or parsing time; must be
a code in the ISO 639-1 2 character language code-set.

h|prop +
  
|*concept_code*: xref:representation:builtin_types-gen.adoc#_class_string[String]
a|Term code defining the meaning of the artefact as a whole, and always used as the at-code
(at-coded archetypes) or id-code (id-coded archetypes) on the root node of the artefact.
Must be defined in the `term_definitions` property.

h|prop +
  
|*term_definitions*: xref:representation:builtin_types-gen.adoc#_class_map[Map]<xref:representation:builtin_types-gen.adoc#_class_string[String],xref:representation:builtin_types-gen.adoc#_class_map[Map]<xref:representation:builtin_types-gen.adoc#_class_string[String],xref:languages:aom2-gen.adoc#_class_archetype_term[ARCHETYPE_TERM]>>
a|Directory of term definitions as a two-level table. The outer hash keys are language
codes, e.g. `"en"`, `"de"`, while the inner hash keys are term codes, e.g. `"at0017"`,
`"at0004"` (at-coded archetypes) or `"id17"`, `"at4"` (id-coded archetypes).

h|prop +
  0..1
|*term_bindings*: xref:representation:builtin_types-gen.adoc#_class_map[Map]<xref:representation:builtin_types-gen.adoc#_class_string[String],xref:representation:builtin_types-gen.adoc#_class_map[Map]<xref:representation:builtin_types-gen.adoc#_class_string[String],xref:representation:builtin_types-gen.adoc#_class_uri[Uri]>>
a|Directory of bindings to external terminology codes and value sets, as a two-level table.
The outer hash keys are terminology ids, e.g. `"SNOMED_CT"`, and the inner hash keys are
constraint codes, e.g. `"at0004"`, `"at4"`, "ac13"` or paths. The indexed `Uri` objects
represent references to externally defined resources, either terms, ontology concepts, or
terminology subsets / ref-sets.

h|prop +
  
|*owner_archetype*: xref:languages:aom2-gen.adoc#_class_archetype[ARCHETYPE]
a|Archetype that owns this terminology.

h|prop +
  0..1
|*value_sets*: xref:representation:builtin_types-gen.adoc#_class_map[Map]<xref:representation:builtin_types-gen.adoc#_class_string[String],xref:languages:aom2-gen.adoc#_class_value_set[VALUE_SET]>
a|Archetype-local value sets, each keyed by value-set id, i.e. an ac-code.

h|prop +
  0..1
|*terminology_extracts*: xref:representation:builtin_types-gen.adoc#_class_map[Map]<xref:representation:builtin_types-gen.adoc#_class_string[String],xref:representation:builtin_types-gen.adoc#_class_map[Map]<xref:representation:builtin_types-gen.adoc#_class_string[String],xref:languages:aom2-gen.adoc#_class_archetype_term[ARCHETYPE_TERM]>>
a|Directory of extracts of external terminologies, as a two-level table. The outer hash keys
are terminology ids, e.g. `"SNOMED_CT"`, while the inner hash keys are term codes or code-
phrases from the relevant terminology, e.g. `"10094842"`.

h| func
|*specialisation_depth* : xref:representation:builtin_types-gen.adoc#_class_integer[Integer]
a|Specialisation depth of this artefact. Unspecialised artefacts have depth 0, with each
additional level of specialisation adding 1 to the specialisation_depth.

h| func
|*node_codes* : xref:representation:builtin_types-gen.adoc#_class_list[List]<xref:representation:builtin_types-gen.adoc#_class_string[String]>
a|For at-coded archetypes: list of all at codes in the terminology., i.e. the 'at' codes in
an ADL archetype, which are the `node_id` values on `C_OBJECT` descendants. For id-coded
archetypes: list of all id codes in the terminology., i.e. the 'id' codes in an ADL
archetype, which are the `node_id` values on `C_OBJECT` descendants.

h| func
|*value_codes* : xref:representation:builtin_types-gen.adoc#_class_list[List]<xref:representation:builtin_types-gen.adoc#_class_string[String]>
a|List of all value term codes in the terminology, i.e. the 'at' codes in an ADL archetype,
which are used as possible values on terminological constrainer nodes.

h| func
|*value_set_codes* : xref:representation:builtin_types-gen.adoc#_class_list[List]<xref:representation:builtin_types-gen.adoc#_class_string[String]>
a|List of all value set codes in the terminology defining value sets. These correspond to
the 'ac' codes in an ADL archetype.

h| func
|*has_language* ( +
a_lang: xref:representation:builtin_types-gen.adoc#_class_string[String]): xref:representation:builtin_types-gen.adoc#_class_boolean[Boolean]
a|True if language `a_lang` is present in archetype terminology.

h| func
|*has_terminology* ( +
a_terminology_id: xref:representation:builtin_types-gen.adoc#_class_string[String]): xref:representation:builtin_types-gen.adoc#_class_boolean[Boolean]
a|True if there are bindings to terminology `a_terminology_id` present. True if
terminology `a_terminology' is present in archetype ontology.

h| func
|*has_term_code* ( +
a_code: xref:representation:builtin_types-gen.adoc#_class_string[String]): xref:representation:builtin_types-gen.adoc#_class_boolean[Boolean]
a|True if code `a_code` defined in this terminology.

h| func
|*term_definition* ( +
a_lang: xref:representation:builtin_types-gen.adoc#_class_string[String], +
a_code: xref:representation:builtin_types-gen.adoc#_class_string[String]): xref:languages:aom2-gen.adoc#_class_archetype_term[ARCHETYPE_TERM] +
 +
  *pre_cond* +
    valid_code: has_term_definition(a_lang, a_code);

a|Term definition for a code, in a specified language.

h| func
|*term_binding* ( +
a_terminology: xref:representation:builtin_types-gen.adoc#_class_string[String], +
a_code: xref:representation:builtin_types-gen.adoc#_class_string[String]): xref:representation:builtin_types-gen.adoc#_class_uri[Uri] +
 +
  *pre_cond* +
    valid_code: has_term_binding(a_terminology_id, a_code);

a|Binding of constraint corresponding to `a_code` in target external terminology
`a_terminology_id`, as a string, which is usually a formal query expression.

h| func
|*terminologies_available* : xref:representation:builtin_types-gen.adoc#_class_list[List]<xref:representation:builtin_types-gen.adoc#_class_string[String]>
a|List of terminologies to which term or constraint bindings exist in this terminology,
computed from bindings.

h| func
|*terminology_extract_term* ( +

 +

h| func
