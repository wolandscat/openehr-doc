3+^h|  *class C_DURATION*

h|*Description*
2+a|Constraint on instances representing duration, which is assumed to have the same parts as
the ISO 8601 duration string. Thus, constraints are of the form `"PWD"` (weeks and/or
days), `"PDTHMS"` (days, hours, minutes, seconds) and so on.

Both range and the constraint pattern can be set at the same time, corresponding to the
ADL constraint `"PWD/\|;P0W..P50W\|;"`.

As for all of openEHR, two ISO 8601 exceptions are allowed:

* the 'W' (week) designator can be mixed in - the allowed patterns are:
`P[Y\|;y][M\|;m][D\|;d][T[H\|;h][M\|;m][S\|;s]]` and `P[W\|;w]`;
* the values used in an interval constraint may be negated, i.e. a leading minus (`'-'`)
sign may be used.
