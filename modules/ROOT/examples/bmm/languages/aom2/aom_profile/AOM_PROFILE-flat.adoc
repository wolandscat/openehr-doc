3+^h|  *class AOM_PROFILE*

h|*Description*
2+a|Profile of common settings relating to use of reference model(s) and terminology for a
given archetype developing organisation.

h|*Inherit*
2+|xref:representation:builtin_types-gen.adoc#_class_any[Any]

h|
2+^h|*Feature Group "access"*

h|prop +
  
|*profile_name*: xref:representation:builtin_types-gen.adoc#_class_string[String]
a|Name of this profile, usually based on the publisher it pertains to e.g. "openEHR",
"CDISC", etc.

h|prop +
  0..1
|*archetype_parent_class*: xref:representation:builtin_types-gen.adoc#_class_string[String]
a|States a class from the Reference Model that provides archetyping capability in RM data
structures. This attribute is optional, and there need be no such class in the RM.
Defining it here has the effect in tools that the class tree under which archetypes are
arranged contains only RM classes inheriting from this class.

h|prop +
  0..1
|*archetype_data_value_parent_class*: xref:representation:builtin_types-gen.adoc#_class_string[String]
a|This attribute defines a base class from the Reference Model whose descendants are
considered to be 'logical data types', i.e. of some higher level than the built-in
primitive types String, Integer etc. This attribute is optional, even if the RM does have
such a class, and is only used to help tooling to provide more intelligent display.

h|prop +
  0..1
|*archetype_visualise_descendants_of*: xref:representation:builtin_types-gen.adoc#_class_string[String]
a|If `archetype_parent_class` is not set, designate a class whose descendants should be
made visible in tree and grid renderings of the archetype definition.

h|prop +
  0..1
|*aom_rm_type_mappings*: xref:representation:builtin_types-gen.adoc#_class_map[Map]<xref:representation:builtin_types-gen.adoc#_class_string[String],xref:languages:aom2-gen.adoc#_class_aom_type_mapping[AOM_TYPE_MAPPING]>
a|Mappings from AOM built-in types to actual types in RM: whenever the type name is
encountered in an archetype, it is mapped to a specific RM type.

h|prop +
  0..1
|*aom_rm_type_substitutions*: xref:representation:builtin_types-gen.adoc#_class_map[Map]<xref:representation:builtin_types-gen.adoc#_class_string[String],xref:representation:builtin_types-gen.adoc#_class_string[String]>
a|Allowed type substitutions: Actual RM type names keyed by AOM built-in types which can
substitute for them in an archetype. E.g. `<key = "ISO8601_DATE", value = "String">` means
that if RM property `TYPE._some_property` is of type `String`, an `ISO8601_DATE` is
allowed at that position in the archetype.

h|prop +
  0..1
|*aom_lifecycle_mappings*: xref:representation:builtin_types-gen.adoc#_class_map[Map]<xref:representation:builtin_types-gen.adoc#_class_string[String],xref:representation:builtin_types-gen.adoc#_class_string[String]>
a|List of mappings of lifecycle state names used in archetypes to AOM lifecycle state names:

* key = source lifecycle state;
* value = AOM lifecycle state.

h|prop +
  0..1
|*rm_primitive_type_equivalences*: xref:representation:builtin_types-gen.adoc#_class_map[Map]<xref:representation:builtin_types-gen.adoc#_class_string[String],xref:representation:builtin_types-gen.adoc#_class_string[String]>
a|Equivalences of RM primitive types to in-built set of primitive types. Used to determine
which AOM `C_PRIMITIVE_OBJECT` descendant is used for a primitive type. Typical entries:

* `value`       `key`
* "Real"        "Double"
* "Integer"     "Integer64"
