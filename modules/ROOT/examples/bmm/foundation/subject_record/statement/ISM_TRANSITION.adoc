3+^h|  *class ISM_TRANSITION*

h|*Description*
2+a|Model of a transition in the Instruction State Machine, caused by a careflow step.
The attributes document the careflow step as well as the ISM transition.

h|*Inherit*
2+|xref:representation:representation-data_marking.adoc#_class_pathable[PATHABLE]

h|
2+^h|*Feature Group "access"*

h|prop +
  
|*current_state*: xref:representation:representation-data_types.adoc#_class_dv_coded_text[DV_CODED_TEXT]
a|The ISM current state. Coded by openEHR terminology group Instruction states.

h|prop +
  0..1
|*transition*: xref:representation:representation-data_types.adoc#_class_dv_coded_text[DV_CODED_TEXT]
a|The ISM transition which occurred to arrive in the current_state. Coded by
openEHR terminology group Instruction transitions.

h|prop +
  0..1
|*careflow_step*: xref:representation:representation-data_types.adoc#_class_dv_coded_text[DV_CODED_TEXT]
a|The step in the careflow process which occurred as part of generating this action,
e.g. dispense, start_administration. Represents the clinical label for the
activity, as opposed to current_state which represents the ISM computable form.

h|prop +
  0..1
|*reason*: xref:representation:representation-builtin_types.adoc#_class_list[List]<xref:representation:representation-data_types.adoc#_class_dv_text[DV_TEXT]>
a|Optional possibility of adding one or more reasons for this careflow step having
been taken.
