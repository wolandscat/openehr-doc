3+^h|  *class ISM_TRANSITION*

h|*Description*
2+a|Model of a transition in the Instruction State Machine, caused by a careflow step.
The attributes document the careflow step as well as the ISM transition.

h|*Inherit*
2+|xref:representation:data_marking-gen.adoc#_class_pathable[PATHABLE]

h|
2+^h|*Feature Group "access"*

h|prop +
  
|*current_state*: xref:representation:data_types-gen.adoc#_class_dv_coded_text[DV_CODED_TEXT]
a|The ISM current state. Coded by openEHR terminology group Instruction states.

h|prop +
  0..1
|*transition*: xref:representation:data_types-gen.adoc#_class_dv_coded_text[DV_CODED_TEXT]
a|The ISM transition which occurred to arrive in the current_state. Coded by openEHR
terminology group Instruction transitions.

h|prop +
  0..1
|*careflow_step*: xref:representation:data_types-gen.adoc#_class_dv_coded_text[DV_CODED_TEXT]
a|The step in the careflow process which occurred as part of generating this action, e.g.
dispense , start_administration. This attribute represents the clinical label for the
activity, as opposed to current_state which represents the state machine (ISM) computable
form. Defined in archetype.

h|prop +
  0..1
|*reason*: xref:representation:builtin_types-gen.adoc#_class_list[List]<xref:representation:data_types-gen.adoc#_class_dv_text[DV_TEXT]>
a|Optional possibility of adding one or more reasons for this careflow step having been
taken. Multiple reasons may occur in medication management for example.

h|*Invariants*
2+a|

h|
2+a|current_state_valid: terminology(Terminology_id_openehr).has_code_for_group_id(Group_id_instruction_states, current_state.defining_code);

h|
2+a|transition_valid: ∃ transition ⇒ terminology(Terminology_id_openehr).has_code_for_group_id(Group_id_instruction_transitions, transition.defining_code);
