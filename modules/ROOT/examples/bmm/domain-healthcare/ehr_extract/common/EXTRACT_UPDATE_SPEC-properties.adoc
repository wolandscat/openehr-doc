|*repeat_period*: xref:representation:data_types-gen.adoc#_class_dv_duration[DV_DURATION]
a|Period for resending update Extracts in response to original Request.

h|prop +
  0..1
|*trigger_events*: xref:representation:builtin_types-gen.adoc#_class_list[List]<xref:representation:data_types-gen.adoc#_class_dv_coded_text[DV_CODED_TEXT]>
a|Set of Event names that will cause sending of update Extracts. Event types include:

* `\|any_change\|` - any change in content items matched by content specification,
e.g. new versions of persistent compositions. If the content list allows matching of any,
or a wide range of archetypes, this event type will match any additions to the record.
* `\|correction\|` - match error corrections only, including deletions.
* `\|update\|` - match updates (i.e. new versions) of included content items.

Coded by openEHR Terminology group 'extract update trigger event type'.

h|prop +
  
|*update_method*: xref:representation:data_types-gen.adoc#_class_code_phrase[CODE_PHRASE]
a|Indicate mode of update. Can be: send only items that are changed (including logical
deletions) or new since last send. For `persist_in_server` Requests only.
