3+^h|  *class EXTRACT_SPEC*

h|*Description*
2+a|Specification of an Extract's contents. Subtypes can be used to add details specific to
the type of Extract. The specification consists of attributes specifying the directory,
and two further groups of attributes in their own classes, namely a version specfication
(which versions of information items are to be included) and a manifest (which entities
are to be included in the extract).

Use: Used in a request to specify an Extract, as well as to describe what is contained in
an Extract.

h|*Inherit*
2+|xref:representation:builtin_types-gen.adoc#_class_any[Any]

h|
2+^h|*Feature Group "access"*

h|prop +
  0..1
|*version_spec*: xref:domain-healthcare:ehr_extract-gen.adoc#_class_extract_version_spec[EXTRACT_VERSION_SPEC]
a|Specification of which versions of information items to include in the Extract. If Void,
the default is latest versions only (which is reasonable for non-versioning systems as
well).

h|prop +
  
|*manifest*: xref:domain-healthcare:ehr_extract-gen.adoc#_class_extract_manifest[EXTRACT_MANIFEST]
a|Specification of entities (e.g. records) to include in the Extract.

h|prop +
  
|*extract_type*: xref:representation:data_types-gen.adoc#_class_dv_coded_text[DV_CODED_TEXT]
a|Coded term indicating the type of content required, e.g.

* `\|openehr-ehr\|`
* `\|openehr-demographic\|`
* `\|generic-emr\|`
* `\|other\|`

Coded by openEHR Terminology group `'extract content type'`.

h|prop +
  
|*include_multimedia*: xref:representation:builtin_types-gen.adoc#_class_boolean[Boolean]
a|Indicates whether in-line instances of `DV_MULTIMEDIA` in the source data are included or
not.

h|prop +
  
|*priority*: xref:representation:builtin_types-gen.adoc#_class_integer[Integer]
a|Requested priority of this request to be handled by server. Priority schemes are likely to
be local, and use values agreed by both ends.

TBD: alternative is standard coded terms

h|prop +
  
|*link_depth*: xref:representation:builtin_types-gen.adoc#_class_integer[Integer]
a|Degree of links to follow emanating from content items specified for inclusion. The kind
of links to follow is dependent on the type of Extract.

All items at the target end of followed links at the given depth are also included in the
extract; `EXTRACT_CONTENT_ITEM._is_primary` is used to differentiate.

* 0 = don't follow;
* 1 = follow first degree links;
* 2 = follow 2nd degree links;
* ....
* n = follow nth degree links

h|prop +
  0..1
|*criteria*: xref:representation:builtin_types-gen.adoc#_class_list[List]<xref:representation:data_types-gen.adoc#_class_dv_coded_text[DV_CODED_TEXT]>
a|Queries specifying the contents of this Extract.

h|prop +
  0..1
|*other_details*: xref:representation:data_structures-gen.adoc#_class_item_structure[ITEM_STRUCTURE]
a|Other specification items. Archetypable.
