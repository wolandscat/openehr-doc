3+^h|  *class ATTESTATION*

h|*Description*
2+a|Record an attestation by a party (the committer) to item(s) of record content.
Used for authorisation of controlled substances, witnessing of content, or
indicating acknowledgement of content by an intended recipient.

h|*Inherit*
2+|xref:infostructure:infostructure-versioning.adoc#_class_audit_details[AUDIT_DETAILS]

h|
2+^h|*Feature Group "access"*

h|prop +
  0..1
|*attested_view*: xref:representation:representation-data_types.adoc#_class_dv_multimedia[DV_MULTIMEDIA]
a|Optional visual representation of content attested e.g. screen image.

h|prop +
  0..1
|*proof*: xref:representation:representation-builtin_types.adoc#_class_string[String]
a|Proof of attestation.

h|prop +
  0..1
|*items*: xref:representation:representation-builtin_types.adoc#_class_list[List]<xref:representation:representation-data_types.adoc#_class_dv_ehr_uri[DV_EHR_URI]>
a|Items attested, expressed as fully qualified runtime paths to the items in question.
Although not recommended, these may include fine-grained items which have been attested in
some other system. Otherwise it is assumed to be for the entire VERSION with which it is
associated.

h|prop +
  
|*reason*: xref:representation:representation-data_types.adoc#_class_dv_text[DV_TEXT]
a|Reason of this attestation. Optionally coded by the openEHR Terminology group attestation
reason ; includes values like authorisation , witness etc.

h|prop +
  
|*is_pending*: xref:representation:representation-builtin_types.adoc#_class_boolean[Boolean]
a|True if this attestation is outstanding; False means it has been completed.

h|prop +
  
|*system_id*: xref:representation:representation-builtin_types.adoc#_class_string[String]
a|Identifier of the logical EHR system where the change was committed. This is almost always
owned by the organisation legally responsible for the EHR, and is distinct from any
application, or any hosting infrastructure.

h|prop +
  
|*time_committed*: xref:representation:representation-data_types.adoc#_class_dv_date_time[DV_DATE_TIME]
a|Time of committal of the item.

h|prop +
  
|*change_type*: xref:representation:representation-data_types.adoc#_class_dv_coded_text[DV_CODED_TEXT]
a|Type of change. Coded using the openEHR Terminology audit change type group.

h|prop +
  0..1
|*description*: xref:representation:representation-data_types.adoc#_class_dv_text[DV_TEXT]
a|Reason for committal. This may be used to qualify the value in the `_change_type_` field.
For example, if the change affects only the EHR directory, this field might be used to
indicate 'Folder "episode 2018-02-16" added' or similar.

h|prop +
  
|*committer*: xref:representation:representation-data_marking.adoc#_class_party_proxy[PARTY_PROXY]
a|Identity and optional reference into identity management service, of user who committed
the item.

2+a|
