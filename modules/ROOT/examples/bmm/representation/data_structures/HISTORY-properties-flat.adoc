3+^h|  *class HISTORY<T:ITEM_STRUCTURE>*

h|*Description*
2+a|Root object of a linear history, i.e. time series structure. Generic class whose type parameter
must be a descendant of ITEM_STRUCTURE.

h|*Inherit*
2+|xref:representation:representation-data_structures.adoc#_class_data_structure[DATA_STRUCTURE]

h|
2+^h|*Feature Group "access"*

h|prop +
  
|*origin*: xref:representation:representation-data_types.adoc#_class_dv_date_time[DV_DATE_TIME]
a|Time origin of this event history. The first event is not necessarily at the origin point.

h|prop +
  0..1
|*period*: xref:representation:representation-data_types.adoc#_class_dv_duration[DV_DURATION]
a|Period between samples in this segment if periodic.

h|prop +
  0..1
|*duration*: xref:representation:representation-data_types.adoc#_class_dv_duration[DV_DURATION]
a|Duration of the entire History; either corresponds to the duration of all the events,
and/or the duration represented by the summary, if it exists.

h|prop +
  0..1
|*summary*: xref:representation:representation-data_structures.adoc#_class_item_structure[ITEM_STRUCTURE]
a|Optional summary data that aggregates, organizes, reduces and transforms the event series.
This may be a text or image that presents a graphical presentation, or some data that
assists with the interpretation of the data.

h|prop +
  0..1
|*events*: xref:representation:representation-builtin_types.adoc#_class_list[List]<xref:representation:representation-data_structures.adoc#_class_event[EVENT]<T>>
a|The events in the series. This attribute is of a generic type whose parameter must be a
descendant of `ITEM_SUTRUCTURE`.

h|prop +
  
|*name*: xref:representation:representation-data_types.adoc#_class_dv_text[DV_TEXT]
a|Runtime name of this fragment, used to build runtime paths. This is the term provided via
a clinical application or batch process to name this EHR construct: its retention in the
EHR faithfully preserves the original label by which this entry was known to end users.

h|prop +
  0..1
|*links*: xref:representation:representation-builtin_types.adoc#_class_list[List]<xref:representation:representation-data_marking.adoc#_class_link[LINK]>
a|Links to other archetyped structures (data whose root object inherits from `ARCHETYPED`,
such as `ENTRY`, `SECTION` and so on). Links may be to structures in other compositions.

h|prop +
  0..1
|*feeder_audit*: xref:representation:representation-data_marking.adoc#_class_feeder_audit[FEEDER_AUDIT]
a|Audit trail from non-openEHR system of original commit of information forming the content
of this node, or from a conversion gateway which has synthesised this node.

h|
2+^h|*Feature Group "infrastructure"*

h|prop +
  0..1
|*uid*: xref:representation:representation-identification.adoc#_class_uid_based_id[UID_BASED_ID]
a|Optional globally unique object identifier for root points of archetyped structures.

h|prop +
  
|*archetype_node_id*: xref:representation:representation-builtin_types.adoc#_class_string[String]
a|Design-time archetype identifier of this node taken from its generating archetype; used to
build archetype paths. Always in the form of an at-code, e.g. `at0005`. This value enables
a 'standardised' name for this node to be generated, by referring to the generating
archetype local terminology.

At an archetype root point, the value of this attribute is always the stringified form of
the `_archetype_id_` found in the `_archetype_details_` object.

h|prop +
  0..1
|*archetype_details*: xref:representation:representation-data_marking.adoc#_class_archetyped[ARCHETYPED]
a|Details of archetyping used on this node.

h|*Invariants*
2+a|

h|

h|
2+a|periodic_validity: is_periodic ⊻ ¬ ∃ period;

h|
