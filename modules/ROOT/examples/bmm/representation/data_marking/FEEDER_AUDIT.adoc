3+^h|  *class FEEDER_AUDIT*

h|*Description*
2+a|The `FEEDER_AUDIT` class defines the semantics of an audit trail which is constructed
to describe the origin of data that have been transformed into openEHR form and committed
to the system.

h|
2+^h|*Feature Group "access"*

h|prop +
  0..1
|*originating_system_item_ids*: xref:representation:representation-builtin_types.adoc#_class_list[List]<xref:representation:representation-data_types.adoc#_class_dv_identifier[DV_IDENTIFIER]>
a|Identifiers used for the item in the originating system, e.g. filler and placer ids.

h|prop +
  0..1
|*feeder_system_item_ids*: xref:representation:representation-builtin_types.adoc#_class_list[List]<xref:representation:representation-data_types.adoc#_class_dv_identifier[DV_IDENTIFIER]>
a|Identifiers used for the item in the feeder system, where the feeder system is distinct
from the originating system.

h|prop +
  0..1
|*original_content*: xref:representation:representation-data_types.adoc#_class_dv_encapsulated[DV_ENCAPSULATED]
a|Optional inline inclusion of or reference to original content corresponding to the openEHR
content at this node. Typically a URI reference to a document or message in a persistent
store associated with the EHR.

h|prop +
  
|*originating_system_audit*: xref:representation:representation-data_marking.adoc#_class_feeder_audit_details[FEEDER_AUDIT_DETAILS]
a|Any audit information for the information item from the originating system.

h|prop +
  0..1
|*feeder_system_audit*: xref:representation:representation-data_marking.adoc#_class_feeder_audit_details[FEEDER_AUDIT_DETAILS]
a|Any audit information for the information item from the feeder system, if different from
the originating system.
