h|prop +
  
|*namespace*: xref:representation:builtin_types-gen.adoc#_class_string[String]
a|Namespace to which this identifier belongs in the local system context (and possibly in
any other openEHR compliant environment) e.g. terminology , demographic . These names are
not yet standardised. Legal values for `namespace` are:

* `"local"`
* `"unknown"`
* a string matching the standard regex `[a-zA-Z][a-zA-Z0-9_.:\/&?=+-]*`.

Note that the first two are just special values of the regex, and will be matched by it.

h|prop +
  
|*type*: xref:representation:builtin_types-gen.adoc#_class_string[String]
a|Name of the class (concrete or abstract) of object to which this identifier type refers,
e.g. `PARTY`, `PERSON`, `GUIDELINE` etc. These class names are from the relevant reference
model. The type name `ANY` can be used to indicate that any type is accepted (e.g. if the
type is unknown).

h|prop +
  
|*id*: xref:representation:identification-gen.adoc#_class_object_id[OBJECT_ID]
a|Globally unique id of an object, regardless of where it is stored.
