3+^h|  *class DV_STATE*

h|*Description*
2+a|For representing state values which obey a defined state machine, such as a variable
representing the states of an instruction or care process.

DV_STATE is expressed as a String but its values are driven by archetype-defined state
machines. This provides a powerful way of capturing stateful complex processes in simple
data.

h|*Inherit*
2+|xref:representation:representation-data_types.adoc#_class_data_value[DATA_VALUE]

h|
2+^h|*Feature Group "access"*

h|prop +
  
|*value*: xref:representation:representation-data_types.adoc#_class_dv_coded_text[DV_CODED_TEXT]
a|The state name. State names are determined by a state/event table defined in archetypes,
and coded using openEHR Terminology or local archetype terms, as specified by the
archetype.

h|prop +
  
|*is_terminal*: xref:representation:representation-builtin_types.adoc#_class_boolean[Boolean]
a|Indicates whether this state is a terminal state, such as "aborted", "completed" etc. from
which no further transitions are possible.
