3+^h|  *class CODE_PHRASE*

h|*Description*
2+a|A fully coordinated term from a terminology service.

h|
2+^h|*Feature Group "access"*

h|prop +
  
|*terminology_id*: xref:representation:representation-identification.adoc#_class_terminology_id[TERMINOLOGY_ID]
a|Identifier of the distinct terminology from which the code_string (or its elements) was
extracted.

h|prop +
  
|*code_string*: xref:representation:representation-builtin_types.adoc#_class_string[String]
a|The key used by the terminology service to identify a concept or coordination of concepts.
This string is most likely parsable inside the terminology service, but nothing can be
assumed about its syntax outside that context.

h|prop +
  0..1
|*preferred_term*: xref:representation:representation-builtin_types.adoc#_class_string[String]
a|Optional attribute to carry preferred term corresponding to the code or expression in
`_code_string_`. Typical use in integration situations which create mappings, and
representing data for which both a (non-preferred) actual term and a preferred term are
both required.

h|*Invariants*
2+a|

h|
2+a|code_string_valid: ¬ code_string.is_empty();
